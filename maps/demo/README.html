Demo 地图数据说明

¶编辑模式入口

当前只保留这一种编辑入口：

含义：

  • ?hm=demo
    只让 ID 为 demo 的这张地图进入编辑模式。

补充说明：

  • 只要某个页面里实际使用了 {% hmap ... %} 地图短代码，这个页面通常都可以进入编辑模式。
  • 如果页面里有多张地图，就直接用 ?hm=地图ID 精确指定要编辑的那一张。
  • 这类参数不会在页面上直接显示按钮，所以普通访客一般不会注意到；但它不是严格的权限控制，只是“不明显”。

这份文档用于说明 source/maps/demo/ 目录下各类地图数据的组织方式，方便后续继续增删点、线、面图层，或复制出新的地图目录。

当前目录整理原则是：

  • 根目录只保留入口配置和总说明
  • 正式展示图层放在 layers/
  • 参考数据放在 references/
  • 规则型文本放在 rules/
  • 辅助说明文档放在 docs/

¶目录结构

当前 demo 目录中常用文件大致如下：

各文件用途：

  • meta.json
    地图总配置文件，控制底图、初始视野、缩放范围、图层顺序、标签语言、编辑模式参考层等。
  • layers/places.geojson
    一般地点图层，适合城市、港口、据点、聚落等点位。
  • layers/battles.geojson
    战役或事件点图层。
  • layers/routes.geojson
    路线图层，对应 LineString。
  • layers/regions.geojson
    区域图层，对应 Polygon。
  • references/ancient-greece-major-leaflet.geojson
    编辑模式参考层，目前主要作为古地名辅助参考，不直接作为正式叙事图层。
  • references/pleiades.geojson
    额外参考地名数据，可按需要接入或裁剪。
  • rules/place-display-rules.json
    手工维护的地点显示规则，当前主要包含 core_keep 和 major_keep 两层。
  • docs/leaflet-icon-mapping.md
    当前 Leaflet 图标与地点类型映射表，便于后续统一维护。

¶meta.json 的作用

meta.json 是地图入口配置，前端会优先读取这里。

当前最常用字段如下：

可以这样理解：

  • view.center / view.zoom
    页面初始打开时的中心点与缩放级别。
  • maxBounds
    限制地图平移范围。
  • maxZoom / editMaxZoom
    分别限制正式展示模式与编辑模式的最大缩放。
  • tileLayer
    正式展示模式的底图。
  • editTileLayer
    编辑模式的底图。
  • layers
    正式展示图层列表，顺序就是叠加顺序。
  • editReferenceLayers
    编辑模式的辅助参考层。

¶编辑模式怎么用

当前编辑模式通过 URL 进入：

编辑模式用途：

  • 点击生成点、线、面几何
  • 复制当前几何为 GeoJSON 片段
  • 借助参考层观察古地名位置

当前编辑模式不是自动保存，而是“辅助生成 GeoJSON”。

建议流程：

  1. 打开编辑模式。
  2. 选择点、线、面工具。
  3. 在地图上点击绘制。
  4. 用工具栏复制结果。
  5. 手动粘贴进目标 .geojson 文件。

¶推荐维护方式

后续维护时，建议按语义分文件：

  • 城市、港口、据点：layers/places.geojson
  • 战役、事件：layers/battles.geojson
  • 进军、航线、贸易线：layers/routes.geojson
  • 区域、势力范围：layers/regions.geojson
  • 仅供参考的古地名：references/ancient-greece-major-leaflet.geojson
  • 手工显示规则：rules/place-display-rules.json

这样做的好处：

  • 配色和样式更容易统一
  • 图层逻辑更清楚
  • 后续要隐藏某类信息时更方便

¶新建一张地图的最简单方法

如果以后要新建另一张地图，最简单的方法不是从零开始，而是复制 demo。

建议流程：

  1. 复制 source/maps/demo/ 为新的目录，例如 source/maps/peloponnesian-war/
  2. 修改新的 meta.json
  3. 替换其中各个 .geojson

¶常见错误

¶1. 地图不显示

优先检查：

  • meta.json 是否是合法 JSON
  • layers 里的 file 是否写对相对路径
  • GeoJSON 是否是合法 FeatureCollection

  • labelFields 指向的字段名是否存在
  • labels 是否被设为 false
  • 图层是否真的被加入到 layers

¶4. 编辑模式参考层太卡

参考层数据量过大时，建议：

  • 先裁剪地理范围
  • 只保留主要地名
  • 单独制作一份较小的 major 文件

当前 references/ancient-greece-major-leaflet.geojson 就是这种思路。
